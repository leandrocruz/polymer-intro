Custom, Web Components

A stack of HTML5 technologies that enable higher componentization of web applications:

  1. Custom Elements
  2. Html Imports
  3. Templates
  4. Shadow Dom

See: webcomponents.org

Polymer

Why polymer ?

Polymer is a framework build on top of WebComponents that provides an aditional set of custom components and extra functionality like two way data binding.

See builtwithpolymer.org

Polymer

Seriously, why?

  1. Highly reusable
  2. Declarative approach
  3. Large set of pre-built components
  4. Good documentation/community
  5. Enables good practices
  6. Modern pipeline and tooling
